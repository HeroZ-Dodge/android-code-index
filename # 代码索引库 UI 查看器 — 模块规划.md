

# 代码索引库 UI 查看器 — 模块规划

---

## 一、先确定 UI 的定位

```
这个 UI 的核心目的：

  1. 验证索引质量    → 索引是否正确解析了代码结构
  2. 浏览代码结构    → 快速了解项目全貌（模块、类、函数、资源）
  3. 调试查询接口    → 测试 API 返回的数据是否符合预期
  4. 数据统计总览    → 了解索引覆盖率、各模块规模

  不是：代码编辑器、IDE 替代品、业务文档系统
```

**交互功能：**

| 功能 | 说明 |
|------|------|
| 模块列表（左栏） | 树形结构，可折叠，按模块分组 |
| 模块概要（右栏顶部） | 语言分布、符号总数、依赖模块 |
| 文件/符号树（右栏主体） | 按目录结构展开，展示类 → 函数 → 属性层级 |
| 点击符号 | 展开详情面板：签名、注解、修饰符、所在文件行号 |
| 筛选器 | 按层级（data/domain/presentation）、按类型、按修饰符筛选 |
